## 1. 사전을 한 번 훑어요

네 번째 시간이에요. 이번 시간은 좀 다른 시간이에요. 깊이 파지 않아요. 사전을 한 번 죽 훑어보는 시간이에요. 외우려고 하지 마세요. "어, 이런 게 있구나" 한 번 보시고 가시면 돼요. 두 해 동안 우리가 가장 자주 칠 명령 열 개예요. 한 번 보고 가면 나중에 다시 만났을 때 친근해요.

각 명령마다 짧게 설명하고, 한 번 같이 쳐 보고, 다음 명령으로 넘어가요. 한 명령에 5분 정도 쓸 거예요. 너무 깊이 들어가면 안 돼요. 이건 사전이에요.

## 2. 카탈로그 — 열 개 사전

오늘 만날 열 명을 먼저 한 번 보고 시작해요.

| 번호 | 명령 | 별명 | 하는 일 |
|------|------|------|---------|
| 1 | pwd / ls | 위치 확인 | 지금 어디고, 여기 뭐가 있나 |
| 2 | cd | 이동 | 폴더 사이 옮겨다님 |
| 3 | cat / less | 파일 읽기 | 파일 안의 글자 보기 |
| 4 | top / htop | 살림 보기 | 누가 지금 일을 하고 있나 |
| 5 | df / du | 창고 보기 | 디스크 얼마나 썼나 |
| 6 | ps / kill | 프로세스 관리 | 일하는 친구 보고 멈추기 |
| 7 | ifconfig / ipconfig | 네트워크 신분증 | 내 IP 보기 |
| 8 | ping / curl | 인터넷 시험 | 다른 컴퓨터한테 인사 |
| 9 | date / uptime | 시계 | 지금 몇 시, 켠 지 얼마 |
| 10 | man / --help / history | 매뉴얼 | 모를 때 어디서 찾나 |

이 열 개가 두 해 동안 여러분이 가장 자주 칠 명령들이에요. 천천히 같이 가요.

## 3. 1번 — pwd와 ls

pwd (pwd, "프린트 워킹 디렉토리")는 "지금 어디 있어?"를 묻는 명령이에요. ls (ls, "엘에스")는 "여기 뭐 있어?"를 묻는 명령이에요. 이 두 개가 짝이에요. 어디 가서 어디 있는지 모르면 일을 못 시작하잖아요.

> ▶ **같이 쳐보기**
>
> ```bash
> pwd
> ls
> ls -la
> ```

`pwd`를 치면 `/Users/내이름`처럼 지금 위치가 떠요. 이게 우리 맥에서의 "지금 내 자리"예요. 터미널을 처음 열면 보통 홈 디렉토리, 즉 우리 사용자 폴더에 있어요. `ls`는 거기 있는 파일과 폴더 목록을 보여줘요. `ls -la`는 숨김 파일까지 다 보여줘요. -l은 자세히, -a는 다 보여달라는 뜻이에요. 점(.)으로 시작하는 파일들이 같이 떠요. 그게 숨김 파일이에요. 두 해 후에 우리가 만들 거예요. .gitignore 같은 거.

## 4. 2번 — cd

cd (cd, "체인지 디렉토리")는 "다른 폴더로 가자"는 명령이에요. 우리가 폴더 사이를 옮겨다닐 때 매번 써요.

> ▶ **같이 쳐보기**
>
> ```bash
> cd Desktop
> pwd
> cd ..
> pwd
> cd ~
> pwd
> ```

`cd Desktop`은 바탕화면 폴더로 들어가요. `cd ..`는 한 단계 위로 나가요. 점 두 개가 "위 폴더"라는 뜻이에요. `cd ~`는 어디 있든 홈으로 돌아가요. 물결표(~)는 홈을 가리켜요. 이 세 개만 알면 일단 길 잃을 일은 없어요. 길 잃으면 `cd ~` 한 줄로 집에 와요.

## 5. 3번 — cat과 less

cat (cat, "캣")은 파일 내용을 한 번에 화면에 쏟아내요. less (less, "레스")는 한 화면씩 보여줘요. 짧은 파일은 cat, 긴 파일은 less.

> ▶ **같이 쳐보기**
>
> ```bash
> cat /etc/hosts
> ```

`/etc/hosts`는 우리 맥에 들어 있는 작은 주소록이에요. 보통 짧아요. 한 줄에 IP 주소가 적혀 있고 그 옆에 이름이 적혀 있어요. 자세한 건 3번째 챕터(네트워크 기본)에서 다뤄요. 오늘은 cat이라는 명령이 파일 내용을 보여준다는 사실만 기억하세요.

긴 파일을 볼 땐 `less /var/log/system.log` 같이 써요. 한 화면씩 보고, 스페이스로 다음 화면, q로 나오기.

## 6. 4번 — top과 htop

top (top, "탑")은 우리 맥의 살림을 실시간으로 보여줘요. 누가 시피유를 얼마나 쓰고 있나, 누가 메모리를 얼마나 쓰고 있나, 한 화면에 표로 떠요.

> ▶ **같이 쳐보기**
>
> ```bash
> top
> ```

엔터를 치면 화면이 바뀌면서 실시간 표가 떠요. PID, COMMAND, %CPU, MEM 같은 칸이 보여요. 각 줄이 지금 우리 맥에서 일하고 있는 한 친구예요. 이 친구를 우리는 "프로세스(process)"라고 불러요. 두 해 동안 진짜 자주 만날 단어예요.

빠져나오기는 q를 누르세요. 안 누르면 계속 화면을 차지해요.

htop은 top의 예쁜 버전이에요. 색깔도 들어가고 보기 편해요. 기본 macOS엔 안 깔려 있어서 나중에 6번째 챕터에서 brew라는 도구로 깔 거예요. 오늘은 top만 알아도 충분해요.

## 7. 5번 — df와 du

df (df, "디스크 프리")는 디스크 전체의 여유를 보여주고, du (du, "디스크 유즈")는 폴더가 얼마나 자리를 차지하는지 보여줘요. df는 큰 그림, du는 세부 구역.

> ▶ **같이 쳐보기**
>
> ```bash
> df -h /
> du -sh ~/Downloads
> ```

`df -h /`는 우리 디스크 전체의 사용률을 사람 읽기 좋게 보여줘요. `du -sh ~/Downloads`는 다운로드 폴더가 몇 GB인지 한 줄로 알려줘요. -s는 합계만, -h는 사람 읽기 좋게. 6교시에 디스크 정리할 때 이 둘이 짝궁이에요.

## 8. 6번 — ps와 kill

ps (ps, "피에스")는 지금 일하고 있는 프로세스 목록을 한 번에 보여줘요. top과 비슷한데, top은 실시간이고 ps는 한 번 찍은 사진이에요. kill (kill, "킬")은 그 중 한 친구를 멈추게 해요.

> ▶ **같이 쳐보기**
>
> ```bash
> ps aux | head -20
> ```

aux는 옵션 묶음이에요. 모든 사용자(a)의 모든 프로세스(x)를 자세히(u). 결과가 너무 많으니까 `| head -20`으로 위 20줄만 봐요. 파이프와 head는 자주 쓰는 짝이에요.

kill은 보통 PID라는 번호로 멈춰요. `kill 12345`. 12345는 ps에서 봤던 그 친구의 PID예요. 멈추기는 위험해서 오늘은 시연 안 해요. 7교시 이후에 진짜 필요할 때 다시 봅니다.

## 9. 7번 — ifconfig와 ipconfig

이름이 비슷한데 다른 친구예요. macOS에는 둘 다 있어요. 리눅스에는 ifconfig만 있어요(요즘은 `ip` 명령으로 대체되는 추세).

> ▶ **같이 쳐보기**
>
> ```bash
> ipconfig getifaddr en0
> ifconfig en0 | head -10
> ```

`ipconfig getifaddr en0`는 우리의 와이파이 IP 주소 한 줄. `ifconfig en0`는 와이파이 카드의 자세한 정보 (MAC 주소, IP, 상태 등). en0가 비어 있으면 en1, en2를 시도해 보세요. 어떤 맥은 이더넷이 en0이고 와이파이가 en1이에요.

## 10. 8번 — ping과 curl

ping (ping, "핑")은 다른 컴퓨터한테 "있어?" 하고 물어봐요. curl (curl, "컬")은 다른 컴퓨터한테 진짜 데이터를 한 번 가져와 봐요.

> ▶ **같이 쳐보기**
>
> ```bash
> ping -c 3 google.com
> curl -s -o /dev/null -w "%{http_code}\n" https://google.com
> ```

ping의 -c 3은 세 번만 시도하고 멈추라는 뜻이에요. 안 그러면 무한히 가요. curl의 옵션은 좀 복잡한데, 짧게 말하면 "구글에 한 번 다녀와서 응답 코드만 알려줘"라는 뜻이에요. 200이 뜨면 성공. 두 해 후에 백엔드 만들 때 정말 자주 쓸 명령이에요.

## 11. 9번 — date와 uptime

date (date, "데이트")는 지금 시각, uptime (uptime, "업타임")은 우리 맥이 켜진 지 얼마나 됐는지.

> ▶ **같이 쳐보기**
>
> ```bash
> date
> uptime
> ```

date는 그냥 지금 시각. uptime은 "켜진 지 며칠, 평균 부하" 같은 정보. 평균 부하(load average) 숫자가 1.0 넘으면 시피유가 바쁜 상태예요. 6교시에 다시 봐요.

## 12. 10번 — man, --help, history

마지막은 명령어가 아니라 "모를 때 어디서 찾나"예요. 이게 진짜 중요해요.

> ▶ **같이 쳐보기**
>
> ```bash
> man ls
> ls --help 2>&1 | head -20
> history | tail -20
> ```

man (man, "맨")은 매뉴얼이에요. `man ls`라고 치면 ls의 사용법이 책처럼 떠요. q로 나와요. macOS의 일부 명령은 `--help`도 지원해요. 짧게 사용법만 보여줘요. 마지막 history는 우리가 최근에 친 명령 목록이에요. "어, 아까 그 명령 뭐였더라" 할 때 history로 찾아요.

자, 열 명 인사 끝. 이제 사이드 메뉴로 다섯 명 더, 그리고 우리가 명령들을 어떻게 합쳐 쓰는지 두 가지 마법을 배워요. 그 다음 위험 경고와 두 해 후 시나리오, FAQ 순으로 정리해요.

## 13. 보너스 다섯 — cp·mv·rm·mkdir·touch

열 명 사전에는 못 들어갔지만 매일 만나는 친구들 다섯 명을 짧게 인사하고 가요. 파일을 만들고, 옮기고, 지우는 명령들이에요.

**cp (cp, "씨피")** — 복사. `cp 원본 대상`. `cp note.txt backup.txt` 하면 같은 폴더에 backup.txt가 생겨요. 폴더 통째로 복사하려면 -r 옵션. `cp -r src/ src_backup/`. r은 recursive(재귀적), 즉 폴더 안의 모든 걸 같이 복사하라는 뜻이에요.

**mv (mv, "엠브이")** — 이동 또는 이름 바꾸기. `mv 원본 대상`. `mv old.txt archive/` 하면 archive 폴더로 옮겨요. `mv old.txt new.txt` 하면 같은 폴더에서 이름만 바꿔요. 한 명령이 두 가지 일을 해요. 합리적이에요. "옮기는 거랑 이름 바꾸는 거나 같은 일이지 뭐."

**rm (rm, "알엠")** — 삭제. `rm 파일`. 폴더 삭제는 `rm -r 폴더`. 그리고 휴지통이 없어요. 한 번 지우면 끝이에요. 이게 마지막 §16 위험 경고에서 다시 나올 거예요. 가장 자주 사고치는 명령이에요.

**mkdir (mkdir, "메이크디렉토리")** — 폴더 만들기. `mkdir new_folder`. 여러 단계를 한 번에 만들려면 -p. `mkdir -p a/b/c` 하면 a, a/b, a/b/c가 다 한 번에 생겨요. p는 parents(부모도 같이).

**touch (touch, "터치")** — 빈 파일 만들기 (또는 기존 파일의 시간 갱신). `touch hello.txt` 하면 빈 hello.txt가 생겨요. 두 해 코스에서 새 파일 만들 때 항상 첫 명령이에요.

이 다섯 명을 한 번 같이 쳐보세요.

> ▶ **같이 쳐보기** — 보너스 다섯 한 줄 시연
>
> ```bash
> cd ~
> mkdir -p tmp/cat-vigilante
> cd tmp/cat-vigilante
> touch report.txt
> cp report.txt report-backup.txt
> mv report-backup.txt archive.txt
> ls -la
> rm archive.txt
> ls -la
> cd ~
> rm -r tmp
> ```

10초짜리 작은 의식이에요. 폴더 만들고, 들어가고, 파일 만들고, 복사하고, 옮기고, 보고, 지우고, 나오고, 폴더째로 지우고. 매일 일하는 사이클의 축소판이에요. 한 번 손가락에 익혀두시면 다음 챕터부터 자연스러워요. 마지막의 `rm -r tmp`는 위에서 만든 모든 걸 한 번에 정리하는 거예요. 청소까지가 작업이에요.

## 14. 파이프와 리다이렉션 — 명령을 합치는 마법

지금까지 명령 한 개씩만 봤어요. 이번엔 명령을 묶는 두 가지 기호를 배워요. 파이프 `|` 와 리다이렉션 `>` `>>`. 이 세 기호가 진짜 셸의 마법이에요. 한 번 알고 나면 명령 두 개로 백 가지 일을 할 수 있어요.

**파이프 `|`.** "왼쪽 명령의 출력을 오른쪽 명령의 입력으로 넘겨라." 이미 우리가 몇 번 썼어요. `system_profiler ... | grep "Model"`. system_profiler의 결과를 grep한테 넘기면, grep이 그 중 우리가 원하는 줄만 골라줘요.

> ▶ **같이 쳐보기** — 파이프 두 단 사다리
>
> ```bash
> ps aux | grep "Chrome" | head -5
> ```

세 명령을 두 개의 파이프로 이었어요. ps가 모든 프로세스를 뱉으면, grep이 Chrome 글자가 든 줄만 거르고, head가 그 중 위 다섯 줄만 보여줘요. 사람으로 치면 세 명이 차례로 일하는 공장이에요. 한 사람이 다 할 수 없는 일을, 작은 일을 잘하는 세 명이 줄 서서 처리해요. **유닉스 철학이에요. "한 가지 일을 잘하는 작은 도구들을 파이프로 잇기."** 두 해 후에도 백 번 천 번 만나요.

**리다이렉션 `>` `>>`.** "왼쪽 명령의 출력을 오른쪽 파일에 저장해라." `>` 는 덮어쓰기, `>>` 는 이어쓰기.

> ▶ **같이 쳐보기** — 결과를 파일로
>
> ```bash
> date > today.txt
> cat today.txt
> uptime >> today.txt
> cat today.txt
> rm today.txt
> ```

date의 결과가 today.txt에 저장돼요. cat으로 보면 date 결과 한 줄. 그 다음 uptime을 `>>` 로 이어 쓰면 today.txt 끝에 한 줄 더 붙어요. 두 줄짜리 파일이 돼요. 다 보고 정리는 rm. 두 해 동안 로그 파일 만들 때, 보고서 만들 때, 디버깅할 때 정말 자주 써요.

`>` 와 `>>` 의 차이는 진짜 중요해요. **`>` 는 덮어써요.** 기존 파일이 있으면 다 날리고 새로 써요. 사고 한 번 칠 수 있어요. "어 그 파일 거기 있었는데..." 하고 잃어요. 그래서 처음에는 `>>` (이어쓰기)에 손이 가도록 익숙해지시는 게 안전해요. 진짜 덮어쓰고 싶을 때만 `>` 로 바꾸세요.

여기 한 가지 보너스 기호. `2>&1`. 이게 뭐냐면, 에러 출력(stderr)을 정상 출력(stdout)으로 합치라는 뜻이에요. 두 해 동안 명령이 에러를 내면 그게 화면에 별도로 떠요. 그걸 파일로 같이 저장하고 싶으면 `명령 > out.txt 2>&1` 이렇게 써요. 외우지 마세요. 챕터 노트에 적어두면 충분해요. 6교시 응급처치 시간에 다시 나옵니다.

이 세 기호 (`|` `>` `>>`)가 셸의 진짜 힘이에요. 명령 자체를 외우는 것보다 이 기호 세 개를 손에 익히는 게 두 해 동안 백 배 도움 돼요. 정말로요.

## 15. grep·find·which — 찾기 삼총사

명령어를 다 외울 수 없으니까 우리는 자주 "찾기"를 해요. 세 가지 찾기 명령을 같이 봐요.

**grep (grep, "그렙")** — 글자 찾기. 파이프와 짝. 이미 몇 번 봤죠. `... | grep "찾을 글자"`.

> ▶ **같이 쳐보기**
>
> ```bash
> ls /Applications | grep -i "code"
> ```

`-i` 는 대소문자 무시(ignore case). Visual Studio Code, Code Inspector, 뭐든 code 글자가 든 앱이 다 떠요. `-i` 없으면 정확히 소문자 code만 찾아요.

**find (find, "파인드")** — 파일 찾기. 폴더를 뒤져서 이름이 맞는 파일을 찾아요.

> ▶ **같이 쳐보기**
>
> ```bash
> find ~/Downloads -name "*.pdf" | head -10
> ```

다운로드 폴더에서 .pdf로 끝나는 파일을 다 찾아서 첫 10개만 보여줘요. `*` 은 와일드카드, "아무 글자나 와도 좋다"는 뜻. 두 해 후엔 `find . -name "*.py" -type f` 같은 패턴으로 파이썬 파일을 다 찾는 식으로 자주 써요.

**which (which, "위치")** — 명령어가 어디 있나 찾기.

> ▶ **같이 쳐보기**
>
> ```bash
> which python3
> which git
> which ls
> ```

`which python3` 하면 `/usr/bin/python3` 같은 경로가 떠요. 두 해 후에 파이썬을 여러 버전 깔 거예요(pyenv로). 그때 "지금 내가 쓰는 python3가 어느 폴더의 python3지?" 헷갈릴 때마다 which를 쳐요. 환경 디버깅의 시작이에요.

이 셋이 합쳐지면 진짜 마법이 돼요. 예를 들어 `find . -name "*.md" | xargs grep "고양이"`. 모든 마크다운 파일에서 "고양이" 글자가 든 줄을 다 찾아줘요. xargs는 find의 결과를 grep한테 차례로 넘겨주는 친구예요. 이건 당장 외우지 마세요. "이런 조합이 있다" 정도만요. 6번째 챕터쯤에서 진짜로 만나요.

## 16. 절대 치면 안 되는 다섯 — 위험 명령 경고문

이 섹션은 진짜 무서운 섹션이에요. 두 해 동안 사고 한 번 치면 데이터 다 날아가요. 미리 한 번 보세요. **이 다섯 가지는 절대 치지 마세요.** 장난으로도요.

**🚨 위험 1 — `rm -rf /`**
설명: 루트 디렉토리부터 모든 걸 강제 재귀 삭제. 운영체제 자체를 지워요. 노트북 다시 사야 해요. 농담 아니에요. 인터넷에서 농담 비슷하게 떠도는 명령인데, 진짜 치면 진짜로 다 날아가요. macOS는 좀 보호 장치가 있지만 우회 가능하고, 리눅스 서버에서는 거의 즉사예요.

**🚨 위험 2 — `:(){ :|:& };:`**
설명: 포크 폭탄(fork bomb). 자기 자신을 무한히 복제해서 시피유와 메모리를 초당 수만 개의 프로세스로 채워요. 1초 안에 노트북이 멈춰요. 강제 재부팅만이 답. 호기심에라도 절대 치지 마세요.

**🚨 위험 3 — `dd if=/dev/zero of=/dev/디스크`**
설명: 디스크에 0을 쭉 써서 모든 데이터를 0으로 만들어요. 진짜 깨끗이 지우는 명령이에요. 폐기할 디스크에는 정상 용도이지만, 실수로 우리 메인 디스크에 치면 운영체제와 사진과 코드가 다 0이 돼요. 복구 불가능. 두 해 후 디스크 폐기 챕터에서 안전하게 쓰는 법을 배우는데, 그 전엔 치지 마세요.

**🚨 위험 4 — `chmod -R 777 /` 또는 `chmod -R 000 /`**
설명: 모든 파일의 권한을 한 번에 바꿔요. 777은 누구나 다 읽고 쓰고 실행 가능 (보안 죽음), 000은 아무도 못 함 (시스템 죽음). 둘 다 치면 즉사. 두 해 후 권한 챕터에서 안전한 사용법 배워요.

**🚨 위험 5 — 인터넷에서 복사한 모르는 명령 그냥 붙여넣기**
설명: "이 명령 한 줄이면 도커 설치 끝!" 같은 글에서 명령을 그냥 복사해서 sudo로 치는 행위. 그 안에 위 1~4번 같은 폭탄이 숨겨져 있을 수 있어요. **반드시 한 줄씩 읽고 이해한 후에 치세요.** 모르는 명령은 `man 명령` 으로 한 번 확인. 두 해 코스에서 우리가 매번 강조할 안전 수칙이에요.

이 다섯 가지를 머리에 두시고 가시면, 두 해 동안 데이터 사고를 거의 다 막을 수 있어요. 사고 친 사람의 회복 시간은 평균 며칠이에요. 며칠을 절약하는 가장 큰 방법이에요. 무서워하실 필요는 없어요. 그냥 위 다섯 가지만 안 치시면 일상 명령은 다 안전해요. 우리가 매일 칠 ls, cd, cat, ps 같은 건 100% 안전해요.

## 17. 두 해 후 우리 사이트 — 서버에서 가장 자주 칠 다섯

오늘 배운 명령들이 두 해 후 고양이 자경단 서버 운영에서 어떻게 쓰일지 한 장면 보여드릴게요. 새벽 3시에 알람이 울려요. "사이트가 느려요" 라는 사용자 신고. SSH로 서버에 들어가요. 가장 먼저 칠 다섯 명령은 이거예요.

```bash
uptime          # 1. 평균 부하 확인 — 1.0 넘으면 시피유 바쁨
df -h           # 2. 디스크 가득 찼나 — 95% 넘으면 즉시 정리
free -h         # 3. 램 남았나 — 0에 가까우면 OOM 위험
ps aux --sort=-%cpu | head -10   # 4. 누가 시피유 잡아먹나
ps aux --sort=-%mem | head -10   # 5. 누가 메모리 잡아먹나
```

이 다섯 줄로 90%의 새벽 사고는 원인이 잡혀요. 진짜로요. 두 해 후 SRE(Site Reliability Engineer)라는 직무의 사람들이 매일 새벽에 이 다섯 줄을 쳐요. 우리는 첫 챕터에서 그 다섯 줄의 절반을 이미 배웠어요(uptime, df, ps). 나머지(free, --sort 옵션)도 짝꿍이라 금방 익혀요. 첫 챕터에서 새벽 3시의 응급처치 절반을 이미 손에 넣은 거예요. 작아 보이지만 두 해 후엔 진짜 큰 자산이에요.

그리고 한 가지 더. 두 해 후 우리가 만들 사이트는 자동화된 모니터링 시스템(예: CloudWatch, Datadog, Grafana)이 24시간 위 다섯 줄을 1초마다 체크해요. 사람이 직접 SSH 들어가는 건 정말 비상시뿐이에요. 그래도 "비상시에 손가락이 자동으로 움직여야" 진짜 엔지니어예요. 평소 한 번씩 쳐 보시는 게 그 비상의 보험이에요.

## 18. 자주 묻는 질문 다섯 — 명령어 편

**Q1. "명령어가 너무 많아요. 다 외워야 해요?"**
A. 아니요. 매일 쓰는 건 사실 7~8개예요. ls, cd, pwd, cat, ps, top, df, grep. 나머지는 필요할 때 챕터로 다시 와서 복사하세요. 두 해 동안 자주 쓰는 7~8개는 자연스럽게 손에 붙고, 가끔 쓰는 건 평생 검색해요. 그게 정상이에요. 시니어 개발자도 다 그래요. 진짜로요.

**Q2. "옵션 (-l, -a, -r 같은 것)이 너무 많아요."**
A. 옵션은 명령마다 수십 개 있어요. 아무도 다 안 외워요. 자주 쓰는 조합 (`ls -la`, `ps aux`, `cp -r`, `rm -rf`) 정도만 손에 붙이세요. 나머지는 `man 명령` 또는 `명령 --help`로 그때그때 봐요. 자주 쓰는 5~6개 조합 외에는 평생 검색입니다.

**Q3. "GUI(파인더)랑 명령어 중 어느 게 좋아요?"**
A. 둘 다 써요. 사진 정리 같은 건 파인더가 편하고, 100개 파일 한 번에 이름 바꾸는 건 명령어가 100배 빨라요. 좋은 개발자는 둘을 상황에 맞게 골라요. **편한 도구가 좋은 도구.** 의무감으로 명령어만 쓸 필요 없어요. 도구 신앙은 위험해요.

**Q4. "맥에서 잘 되는데 리눅스 서버에서 안 되는 명령이 있어요."**
A. 자주 있는 일이에요. macOS의 BSD 명령과 리눅스의 GNU 명령이 옵션이 미세하게 달라요. 예: `ls -la` 색깔 옵션, `sed -i` 백업 파일 처리. 두 해 후 우리가 본격적으로 만나는 차이예요. 첫 챕터에서는 "macOS와 리눅스가 90% 같다, 10% 다르다" 정도만 알면 됩니다.

**Q5. "터미널 화면이 망가졌어요. 글자가 깨지거나 명령이 안 먹어요."**
A. `reset` 한 줄 치고 엔터. 안 되면 `clear` 한 줄. 안 되면 터미널 창 닫고 새로 여세요. 그것도 안 되면 노트북 재부팅. 90%는 `reset`에서 풀려요. 무서워하실 일 거의 없어요.

## 18.45. 명령어의 짧은 역사 — 50년 전 사람들의 손가락

오늘 만난 명령들. ls, cd, cat, grep, ps, kill. 이 친구들 나이가 다 50살이 넘어요. 1970년대에 벨 연구소(Bell Labs)에서 유닉스(Unix)라는 운영체제를 만들 때 거의 다 같이 태어났어요. ls는 list의 줄임말, cd는 change directory, cat은 concatenate(이어붙이기), grep은 g/re/p (global/regular expression/print)에서 왔어요. 이름이 짧고 이상한 이유는 1970년대 키보드가 느리고 화면이 작아서예요. 한 글자라도 줄여야 했어요.

50년이 지났는데 왜 안 바뀌었을까요? 두 가지 이유가 있어요. 첫째, **좋은 도구는 안 바뀌어요.** 망치가 100년 전이나 지금이나 비슷한 모양인 것과 같아요. 잘 만든 도구는 새 버전이 필요 없어요. 둘째, **호환성이 너무 중요해요.** ls 한 명령이 전 세계 수십억 대의 컴퓨터에서 매일 수조 번 호출돼요. 이름을 바꾸면 수많은 프로그램이 깨져요. 그래서 이름은 영원히 그대로예요. 옵션과 안쪽은 조용히 발전하지만요.

이게 우리한테 좋은 소식이에요. **첫 챕터에서 배운 명령들이 두 해 후에도, 50년 후에도 같은 이름으로 살아 있을 거예요.** 평생 자산이에요. 다른 분야의 도구는 1년이면 바뀌어요. 자바스크립트 프레임워크는 6개월마다 트렌드가 바뀐다고 하잖아요. 그런데 셸 명령은 50년이 같아요. 그래서 시니어 개발자들이 셸을 사랑해요. 한 번 익힌 게 평생 가니까요.

한 가지 더. 50년 전 사람들이 우리랑 같은 손가락으로 같은 명령을 쳤다고 생각해 보세요. 우리가 오늘 쳐 본 `ls -la` 한 줄이 1971년의 켄 톰슨(Ken Thompson)도 매일 쳤던 한 줄이에요. 우리는 50년 전 컴퓨터 과학자들과 같은 동작을 하고 있어요. 어떻게 보면 작은 영광이에요. 두 해 코스 첫 시간의 명령들이 그렇게 무거운 역사를 등에 지고 있어요. 한 번씩 그 무게를 느끼면서 쳐 보세요. 좀 다른 기분이 들 거예요.

## 18.5. 키보드 단축키 다섯 — 손가락이 빨라지는 비밀

명령보다 더 자주 쓰는 게 사실 키보드 단축키예요. 다섯 개만 손에 익히시면 터미널 속도가 두 배 돼요.

**↑ (위 화살표)** — 직전 명령 다시 띄우기. 누를 때마다 더 옛날 명령으로 거슬러 올라가요. 사실 history와 같은 일을 한 번 더 빨리 하는 거예요. 매일 백 번 누를 거예요. 진짜로요.

**Tab (탭 키)** — 자동 완성. 파일 이름, 폴더 이름, 명령어 이름을 절반만 치고 Tab 누르면 나머지가 자동으로 완성돼요. `cd Desk` 까지 치고 Tab 누르면 `cd Desktop/`이 돼요. 후보가 여러 개면 두 번 누르세요. 후보 목록이 떠요. 두 해 후엔 Tab 안 누르고 명령 치는 게 어색해요.

**Ctrl+C** — 지금 돌고 있는 명령 강제 정지. ping 같은 무한 명령에서 빠져나올 때, 또는 잘못 친 명령 취소할 때. 누르면 즉시 돌아와요. 일종의 "아, 잘못!" 버튼.

**Ctrl+L** — 화면 깨끗이. clear 명령과 같아요. 화면이 너무 어지러우면 한 번 눌러서 새 출발해요.

**Ctrl+R** — 명령 검색. 누르면 `(reverse-i-search)` 가 떠요. 거기 한 두 글자 치면 옛날에 쳤던 명령 중 그 글자가 들어간 게 떠요. 엔터로 실행, Esc로 편집. 이거 하나가 history + grep을 한 손가락에 묶은 거예요. 두 해 동안 가장 자주 쓸 단축키 1위 후보예요.

이 다섯 개만 익혀도 터미널이 두 배 빠른 도구가 돼요. 외우지 마시고, 한 번 손가락에 시켜 보세요. 일주일이면 자동이에요.

## 18.7. 명령 한 줄의 해부학 — 형식 분해 한 번

명령 하나하나 외우기 전에, 모든 명령의 공통 형식을 한 번 보고 가요. 한 번 외워두면 새 명령 봤을 때도 바로 읽을 수 있어요.

```
명령어  옵션들       인자(들)
ls      -la         /Users/mo/Downloads
   ↑      ↑              ↑
"무엇을" "어떻게"      "어디에/무엇에"
```

**명령어** — 첫 단어. 동사 같은 거예요. ls, cd, cat, grep.

**옵션** — 보통 `-한글자` 또는 `--긴이름` 형식. 동작을 살짝 바꿔요. `-l`은 자세히, `-a`는 다 보여줘, `--help`는 도움말. 옵션끼리는 묶을 수 있어요. `-l -a` = `-la`. 두 해 동안 옵션 묶기 자주 봐요.

**인자** — 명령이 동작할 대상. 파일 이름, 폴더 이름, 검색어, IP 주소. 옵션 뒤에 와요. 인자 없이도 동작하는 명령(pwd, date)도 많아요.

이 세 부분만 머리에 두시면, 처음 보는 명령도 70%는 읽혀요. 예를 들어 `find ~/Downloads -name "*.pdf" -type f` 도 분해해 봐요. 명령어는 find, 인자 1은 `~/Downloads` (어디서), 옵션 1은 `-name "*.pdf"` (이름 패턴), 옵션 2는 `-type f` (파일만, 폴더 제외). 한 번에 읽혀요.

여기에 한 가지 더 추가. **공백이 분리자예요.** 한 칸 띄어 쓰기가 단어를 나눠요. 그래서 파일 이름에 공백이 있으면 큰따옴표로 묶어야 해요. `cat "내 노트.txt"`. 큰따옴표 없으면 셸이 "내"와 "노트.txt"를 두 인자로 봐요. 두 해 동안 한 번씩 사고 칠 수 있어요. 미리 한 번 머리에 두세요. 영어로 파일 이름 짓는 습관이 두 해 동안 사고를 90% 줄여요.

## 18.8. 종료 코드와 `&&` `||` — 명령들의 신호등

명령은 끝날 때마다 작은 신호를 남겨요. 그 신호가 **종료 코드(exit code)** 예요. 0이면 성공, 0이 아니면 실패. 사람한테는 안 보이지만, 셸은 보고 있어요. 직접 한 번 볼게요.

> ▶ **같이 쳐보기** — 종료 코드 보기
>
> ```bash
> ls /
> echo $?
> ls /없는폴더
> echo $?
> ```

`$?` 는 직전 명령의 종료 코드를 담은 변수예요. 첫 ls는 성공해서 0이 나와요. 두 번째 ls는 실패해서 1 또는 2가 나와요(에러 종류에 따라 다름). 외울 건 단 하나. **0은 성공, 0이 아니면 실패.**

이 신호가 왜 중요하냐면, 명령들을 신호등 따라 줄 세울 수 있어서예요. 두 가지 기호로요.

**`&&` (앞이 성공해야 뒤를 실행)** — `mkdir build && cd build && make`. mkdir이 성공해야 cd가 실행되고, cd가 성공해야 make가 실행돼요. 한 단계라도 실패하면 거기서 멈춰요. 안전한 사슬이에요.

**`||` (앞이 실패해야 뒤를 실행)** — `cd /tmp || echo "tmp 폴더 없음"`. cd 실패하면 echo로 알려줘요. 보험 같은 거예요.

이 두 기호로 우리는 셸 위에서 작은 if문을 만들 수 있어요. 두 해 후에 우리가 짤 sysinfo.sh, 빌드 스크립트, 배포 스크립트의 한 줄 한 줄에 다 들어가요. 미리 한 번 손에 익혀두세요.

> ▶ **같이 쳐보기** — &&와 || 짧은 시연
>
> ```bash
> mkdir -p ~/tmp/test && cd ~/tmp/test && touch hello.txt && ls && cd ~ && rm -r ~/tmp/test
> echo "성공!" || echo "실패!"
> ```

첫 줄은 한 번에 6단계가 사슬로 이어져 있어요. 다 성공하면 마지막에 폴더가 정리돼요. 한 단계라도 실패하면 그 다음은 안 가요. 둘째 줄은 echo "성공!"이 항상 성공하니까 || 뒤의 "실패!"는 안 떠요. 종료 코드 0이라서요.

이 신호등 두 개가 두 해 동안 정말 자주 만나요. CI/CD라는 단어를 두 해 후에 만날 텐데, 그게 결국 거대한 `&&` 사슬이에요. "테스트 통과해야 빌드, 빌드 성공해야 배포, 배포 성공해야 알림." 첫 챕터의 두 기호가 두 해 후의 인프라 자동화로 자라나요.

## 18.9. 별칭(alias)과 함수 — 손가락을 줄이는 두 가지 도구

자주 쓰는 긴 명령은 짧은 별명으로 줄일 수 있어요. 그게 **alias(별칭)** 와 **shell function(함수)** 예요. 두 해 동안 자기만의 별칭 모음이 점점 늘어요.

**alias.** 형식: `alias 짧은이름='긴 명령'`.

> ▶ **같이 쳐보기** — 별칭 한 번 만들어 보기
>
> ```bash
> alias ll='ls -la'
> ll
> alias gs='git status'
> alias myinfo='sw_vers && uname -a && df -h /'
> ```

`ll` 한 줄 치면 `ls -la` 가 실행돼요. 두 글자만 치면 되니까 손가락이 편해요. `gs` 는 git status. `myinfo` 는 우리가 직접 만든 종합 신분증 한 줄. 진짜 자주 쓰는 명령은 다 별명으로 만드세요. 일주일이면 손가락이 새 별명에 적응해요.

문제는 alias가 **새 터미널을 열면 사라져요.** 매번 다시 만들어야 해요. 그래서 영구 보존하려면 `~/.zshrc` 파일에 한 줄로 적어두세요. zshrc는 zsh가 시작될 때 자동으로 읽는 설정 파일이에요. 두 해 후엔 여러분의 zshrc 파일이 50줄, 100줄로 자라나 있을 거예요. 그게 본인만의 무기예요.

**함수.** alias로 못 하는 게 있어요. 인자를 받는 명령. 예를 들어 `mkdir 폴더명 && cd 폴더명` 같은 두 단계 명령에 폴더명을 인자로 받고 싶을 때. 함수가 필요해요.

> ▶ **같이 쳐보기** — 함수 정의
>
> ```bash
> mkcd() { mkdir -p "$1" && cd "$1"; }
> mkcd ~/tmp/test2
> pwd
> cd ~ && rm -r ~/tmp/test2
> ```

`mkcd 이름` 한 줄로 폴더 만들고 그 안으로 들어가요. `$1` 은 첫 번째 인자라는 뜻이에요. 두 해 후 본인의 zshrc 파일에 mkcd 같은 작은 함수가 다섯 개쯤 있게 될 거예요. 매일 1초씩 아끼는 작은 도구들이에요.

별칭과 함수는 결국 같은 철학에서 나와요. **자주 쓰는 건 짧게 만들기.** 첫 챕터의 캐시 원칙(자주 쓰는 건 가까이)이 키보드 위에서도 똑같이 적용돼요. 두 해 동안 본인만의 짧은 명령들을 늘려가세요. 그게 본인의 색깔이에요. 모든 시니어 개발자는 자기만의 zshrc를 자랑스러워해요.

## 19. 한 줄 정리 — 머리에 남길 한 가지

오늘 한 시간을 한 줄로.

> **명령어는 외우지 말고 찾는 사람이 되라. 그리고 작은 명령을 파이프로 잇는 사람이 되라.**

두 가지가 들어 있어요. 첫째, 외우지 말고 찾기. man, --help, history, 검색. 이 네 가지 도구가 우리의 진짜 무기예요. 둘째, 명령을 잇기. `|`, `>`, `>>`. 이 세 기호로 한 사람이 못 하는 일을 작은 도구 여럿이 줄 서서 처리해요. 유닉스 50년 철학의 핵심이에요. 두 해 동안 이 한 줄이 머리에 남으면 충분해요. 다른 모든 명령은 이 한 줄에서 가지치기로 자라난 잎사귀들이에요. 줄기를 잡고 가시면 잎은 자연스럽게 따라와요.

## 19.5. 오늘 친 모든 명령 한 장 — Ch001 H4 치트시트

자, 한 시간 동안 명령이 진짜 많이 나왔어요. 한 번 박스 한 장으로 정리해 드릴게요. 이 박스 한 장만 캡처해 두세요.

```bash
# 위치·이동
pwd
ls          ls -la
cd Desktop  cd ..   cd ~

# 파일 읽기·만들기·정리
cat /etc/hosts
less /var/log/system.log     # q로 나오기
touch hello.txt
mkdir -p a/b/c
cp src dst                    cp -r srcdir dstdir
mv old new
rm file                       rm -r dir

# 살림 보기
top                           # q로 나오기
ps aux | head -20
ps aux --sort=-%cpu | head -10
ps aux --sort=-%mem | head -10
df -h /
du -sh ~/Downloads
free -h                       # 리눅스
uptime
date

# 네트워크
ipconfig getifaddr en0
ifconfig en0 | head -10
ping -c 3 google.com
curl -s -o /dev/null -w "%{http_code}\n" https://google.com

# 찾기·보기
grep -i "code"
find ~/Downloads -name "*.pdf"
which python3

# 합치기·신호등
명령1 | 명령2 | 명령3        # 파이프
명령 > out.txt                # 덮어쓰기
명령 >> out.txt               # 이어쓰기
명령 > out.txt 2>&1           # 에러까지
명령1 && 명령2                # 성공 사슬
명령1 || 명령2                # 실패 우회
echo $?                       # 직전 종료 코드

# 도움말
man ls
명령 --help
history | tail -20

# 단축키
↑   Tab   Ctrl+C   Ctrl+L   Ctrl+R

# 별칭·함수 (zshrc에)
alias ll='ls -la'
mkcd() { mkdir -p "$1" && cd "$1"; }
```

이 박스가 두 해 동안 가장 자주 펼쳐 볼 한 장이에요. 인쇄해서 책상 옆에 붙이세요. 한 달이면 절반은 손에 붙고, 1년이면 90%가 자동이에요. 매주 한 번씩 박스 한 번 보고, 하나라도 잊은 게 있으면 별표 치고 그 자리만 한 번 더 쳐 보세요. 별표가 학습이에요(H3에서 한 약속이에요).

그리고 한 가지 더. 이 박스의 명령 중에 빨간 글씨로 표시할 만큼 위험한 건 한 개도 없어요. 다 읽기 전용이거나, rm처럼 본인이 만든 파일만 지우는 정도예요. 마음 놓고 매일 쳐 보세요. §16의 다섯 가지(특히 `rm -rf /`)만 안 치시면 사고 안 나요. 약속이에요.

## 19.7. 명령어를 즐기는 법 — 게임처럼 손에 익히기

명령어 학습이 지루할 수 있어요. 그래서 작은 게임 두 개를 권해 드려요. 두 해 동안 가끔 해보시면 손가락이 즐거워져요.

**게임 1 — 매일 다섯 명 인사 게임.** 매일 아침 터미널 켜자마자 명령 다섯 개를 차례로 치세요. 본인이 정하시면 돼요. 예를 들어 `pwd && ls && uptime && df -h / && date`. 5초 걸려요. 한 달 하면 그 다섯 명령은 숨 쉬듯 자동이에요. 작은 의식이지만 큰 효과예요. 저는 이걸 매일 해요. 컴퓨터한테 인사하는 작은 인사예요.

**게임 2 — alias 한 개씩 늘리기 게임.** 일주일에 한 개씩 본인 zshrc에 alias를 추가하세요. 한 달이면 4개, 1년이면 50개. 두 해면 100개. 100개의 본인만의 명령이 손에 들어와요. 한 번 적은 alias는 본인이 진짜 자주 쓰는 패턴이에요. 그 100개가 본인의 개발자 색깔이에요. 다른 사람의 zshrc를 들여다 보면 그 사람이 어떤 일을 자주 하는지 알 수 있어요. 본인의 zshrc도 점점 본인의 자기소개가 돼요.

이 두 게임이 두 해 동안 명령어와 친구가 되는 가장 빠른 길이에요. 외우려고 끙끙거리지 마시고, 매일 작게 즐겨주세요. 즐거우면 오래 가요. 오래 가면 손에 붙어요. 손에 붙으면 평생 자산이에요. 우리 코스의 작은 비밀이에요.

한 가지만 더 보너스 게임. **게임 3 — 명령 한 줄 시 짓기.** 좀 이상한 게임 같지만 진짜 효과가 있어요. 명령 두세 개를 파이프로 이어서 한 줄로 작은 시처럼 짜 보는 거예요. 예를 들어 `ls ~/Downloads | grep "2026" | wc -l`. 다운로드 폴더에서 2026이 든 파일 개수를 세어 줘요. 한 줄짜리 시예요. 매일 한 줄씩 다른 시를 짜 보세요. SNS에 자랑해도 좋아요. 두 해 후엔 본인이 짠 시 700편이 모여 있어요. 그게 본인의 셸 마스터링의 증거예요.

이 게임 세 개를 같이 하시면 한 달이면 명령어가 친구가 돼요. 약속해요. 처음의 어색함은 한 달이면 사라져요. 그 후엔 즐거움만 남아요. 그 즐거움이 두 해를 가게 해요. 우리 코스의 진짜 동력이에요.

자, 사전을 한 번 죽 훑었어요. 열 개 명령 + 보너스 다섯 + 파이프·리다이렉션 두 마법 + 찾기 삼총사 + 위험 다섯 + 새벽 3시의 다섯 + FAQ + 키보드 단축키 + 명령 해부학 + 신호등 + 별칭·함수 + 50년 역사 + 한 장 치트시트 + 명령 즐기기 게임 셋. 한 시간이 꽤 알찼죠. 외우지 마세요. 한 번 봤다는 사실이 중요해요. 두 해 동안 이 사전의 페이지를 자주 펼치게 될 거예요. 처음에는 매번 챕터 노트로 돌아오시고, 한 달 후엔 절반쯤은 손가락이 기억해요. 1년 후엔 90%가 자동이에요. 두 해 후엔 의식적으로 생각 안 해도 손이 먼저 가요. 그게 진짜 사용자가 되는 길이에요.

그리고 한 가지 약속. 오늘 챕터 끝나기 전에 §13의 보너스 다섯 시연 한 번 더 쳐 보세요. mkdir -p tmp/cat-vigilante부터 rm -r tmp까지요. 손가락에 한 번 익혀두시면 다음 챕터부터 자연스러워요. 매일 한 번씩 이 시연을 일주일만 반복하시면 cp/mv/rm/mkdir/touch가 다 손에 붙어요. 일주일이에요. 짧아요. 약속이에요. 그리고 §19.5의 치트시트 박스를 캡처해서 책상 옆에 붙이시고, 매주 한 번씩 보세요. 이 한 장이 두 해 동안 가장 자주 펼칠 한 장이에요.

그리고 마지막으로 한 가지 더 부탁이에요. 오늘 zshrc 파일에 본인만의 alias 한 줄만 추가해 보세요. 뭐든 좋아요. `alias ll='ls -la'` 도 좋고, `alias myinfo='sw_vers && uname -a'` 도 좋아요. 본인 zshrc 파일의 첫 줄을 본인이 적는 그 순간이 진짜 개발자의 시작이에요. 두 해 후 졸업할 때 zshrc 파일이 100줄로 자라 있을 거예요. 그게 본인만의 색깔이에요. 첫 한 줄을 오늘 적으세요. 그 한 줄이 작은 졸업장이에요. 본인이 본인 환경을 본인이 꾸미기 시작한 첫 흔적이에요. 두 해 후에 그 첫 줄을 다시 보면 진짜 뭉클해져요. 약속해요.

다음 다섯 번째 시간이 오늘의 하이라이트예요. 우리가 직접 sysinfo.sh라는 작은 스크립트 한 개를 같이 짭니다. 자기 맥의 정보를 한 화면에 예쁘게 정리해서 출력해주는 도구예요. 오늘 배운 명령들이 그 안에 다 들어가요. sw_vers, uname, sysctl, df, ipconfig, date 다요. 사전 다음 단계가 진짜 도구 만들기예요. 점심 먹고 와서 시작해요. 한 시간 잘 쉬세요. 점심은 든든히 드세요. 다음 한 시간이 진짜 만드는 시간이라 머리에 에너지가 좀 필요해요. 가능하면 탄수화물보다 단백질 위주로요. 졸리지 않게요. 아, 점심 드시기 전에 오늘 배운 명령 중 하나만 가족이나 친구한테 한 번 자랑해 보세요. "내가 오늘 ps aux로 컴퓨터 안 들여다보는 법 배웠어." 한 마디면 충분해요. 한 번 입으로 풀어내면 진짜 내 것이 돼요. 점심 후에 만나요. 우리 이미 챕터의 절반을 함께 했어요. 박수 한 번 쳐 주세요. 진짜로 잘하고 계세요.

## 👨‍💻 개발자 노트

> - 의도적으로 깊이를 얕게 유지. 각 명령의 본격 학습은 Ch 006(터미널 마스터)에서 진행. 본 카탈로그는 "두 해 동안 다시 만날 친구들의 얼굴 한 번 보기" 목적.
> - `top`/`ps`는 Ch 002 OS 챕터의 프로세스/스레드 본격 학습으로 직결. PID 개념을 여기서 한 번 노출하는 것이 OS 챕터 진입을 부드럽게 합니다.
> - `curl` 한 줄은 Ch 003 네트워크, Ch 030 HTTP, Ch 052 Flask API 디버깅, Ch 110 Lambda 호출 시험에서 거듭 회수됩니다.
> - `ifconfig`는 macOS·구리눅스 명령. 최신 리눅스는 `ip addr`. Ch 003에서 둘의 차이와 마이그레이션을 다룹니다. 본 챕터에서는 "macOS 기준" 단서만.
> - `man`을 마지막에 강조한 것은 의도 — "외우지 말고 찾는 사람이 되라"는 1교시 약속의 행동 지침입니다.
