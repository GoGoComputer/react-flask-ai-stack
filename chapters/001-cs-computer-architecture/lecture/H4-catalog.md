## 1. 사전을 한 번 훑어요

네 번째 시간이에요. 이번 시간은 좀 다른 시간이에요. 깊이 파지 않아요. 사전을 한 번 죽 훑어보는 시간이에요. 외우려고 하지 마세요. "어, 이런 게 있구나" 한 번 보시고 가시면 돼요. 두 해 동안 우리가 가장 자주 칠 명령 열 개예요. 한 번 보고 가면 나중에 다시 만났을 때 친근해요.

각 명령마다 짧게 설명하고, 한 번 같이 쳐 보고, 다음 명령으로 넘어가요. 한 명령에 5분 정도 쓸 거예요. 너무 깊이 들어가면 안 돼요. 이건 사전이에요.

## 2. 카탈로그 — 열 개 사전

오늘 만날 열 명을 먼저 한 번 보고 시작해요.

| 번호 | 명령 | 별명 | 하는 일 |
|------|------|------|---------|
| 1 | pwd / ls | 위치 확인 | 지금 어디고, 여기 뭐가 있나 |
| 2 | cd | 이동 | 폴더 사이 옮겨다님 |
| 3 | cat / less | 파일 읽기 | 파일 안의 글자 보기 |
| 4 | top / htop | 살림 보기 | 누가 지금 일을 하고 있나 |
| 5 | df / du | 창고 보기 | 디스크 얼마나 썼나 |
| 6 | ps / kill | 프로세스 관리 | 일하는 친구 보고 멈추기 |
| 7 | ifconfig / ipconfig | 네트워크 신분증 | 내 IP 보기 |
| 8 | ping / curl | 인터넷 시험 | 다른 컴퓨터한테 인사 |
| 9 | date / uptime | 시계 | 지금 몇 시, 켠 지 얼마 |
| 10 | man / --help / history | 매뉴얼 | 모를 때 어디서 찾나 |

이 열 개가 두 해 동안 여러분이 가장 자주 칠 명령들이에요. 천천히 같이 가요.

## 3. 1번 — pwd와 ls

pwd (pwd, "프린트 워킹 디렉토리")는 "지금 어디 있어?"를 묻는 명령이에요. ls (ls, "엘에스")는 "여기 뭐 있어?"를 묻는 명령이에요. 이 두 개가 짝이에요. 어디 가서 어디 있는지 모르면 일을 못 시작하잖아요.

> ▶ **같이 쳐보기**
>
> ```bash
> pwd
> ls
> ls -la
> ```

`pwd`를 치면 `/Users/내이름`처럼 지금 위치가 떠요. 이게 우리 맥에서의 "지금 내 자리"예요. 터미널을 처음 열면 보통 홈 디렉토리, 즉 우리 사용자 폴더에 있어요. `ls`는 거기 있는 파일과 폴더 목록을 보여줘요. `ls -la`는 숨김 파일까지 다 보여줘요. -l은 자세히, -a는 다 보여달라는 뜻이에요. 점(.)으로 시작하는 파일들이 같이 떠요. 그게 숨김 파일이에요. 두 해 후에 우리가 만들 거예요. .gitignore 같은 거.

## 4. 2번 — cd

cd (cd, "체인지 디렉토리")는 "다른 폴더로 가자"는 명령이에요. 우리가 폴더 사이를 옮겨다닐 때 매번 써요.

> ▶ **같이 쳐보기**
>
> ```bash
> cd Desktop
> pwd
> cd ..
> pwd
> cd ~
> pwd
> ```

`cd Desktop`은 바탕화면 폴더로 들어가요. `cd ..`는 한 단계 위로 나가요. 점 두 개가 "위 폴더"라는 뜻이에요. `cd ~`는 어디 있든 홈으로 돌아가요. 물결표(~)는 홈을 가리켜요. 이 세 개만 알면 일단 길 잃을 일은 없어요. 길 잃으면 `cd ~` 한 줄로 집에 와요.

## 5. 3번 — cat과 less

cat (cat, "캣")은 파일 내용을 한 번에 화면에 쏟아내요. less (less, "레스")는 한 화면씩 보여줘요. 짧은 파일은 cat, 긴 파일은 less.

> ▶ **같이 쳐보기**
>
> ```bash
> cat /etc/hosts
> ```

`/etc/hosts`는 우리 맥에 들어 있는 작은 주소록이에요. 보통 짧아요. 한 줄에 IP 주소가 적혀 있고 그 옆에 이름이 적혀 있어요. 자세한 건 3번째 챕터(네트워크 기본)에서 다뤄요. 오늘은 cat이라는 명령이 파일 내용을 보여준다는 사실만 기억하세요.

긴 파일을 볼 땐 `less /var/log/system.log` 같이 써요. 한 화면씩 보고, 스페이스로 다음 화면, q로 나오기.

## 6. 4번 — top과 htop

top (top, "탑")은 우리 맥의 살림을 실시간으로 보여줘요. 누가 시피유를 얼마나 쓰고 있나, 누가 메모리를 얼마나 쓰고 있나, 한 화면에 표로 떠요.

> ▶ **같이 쳐보기**
>
> ```bash
> top
> ```

엔터를 치면 화면이 바뀌면서 실시간 표가 떠요. PID, COMMAND, %CPU, MEM 같은 칸이 보여요. 각 줄이 지금 우리 맥에서 일하고 있는 한 친구예요. 이 친구를 우리는 "프로세스(process)"라고 불러요. 두 해 동안 진짜 자주 만날 단어예요.

빠져나오기는 q를 누르세요. 안 누르면 계속 화면을 차지해요.

htop은 top의 예쁜 버전이에요. 색깔도 들어가고 보기 편해요. 기본 macOS엔 안 깔려 있어서 나중에 6번째 챕터에서 brew라는 도구로 깔 거예요. 오늘은 top만 알아도 충분해요.

## 7. 5번 — df와 du

df (df, "디스크 프리")는 디스크 전체의 여유를 보여주고, du (du, "디스크 유즈")는 폴더가 얼마나 자리를 차지하는지 보여줘요. df는 큰 그림, du는 세부 구역.

> ▶ **같이 쳐보기**
>
> ```bash
> df -h /
> du -sh ~/Downloads
> ```

`df -h /`는 우리 디스크 전체의 사용률을 사람 읽기 좋게 보여줘요. `du -sh ~/Downloads`는 다운로드 폴더가 몇 GB인지 한 줄로 알려줘요. -s는 합계만, -h는 사람 읽기 좋게. 6교시에 디스크 정리할 때 이 둘이 짝궁이에요.

## 8. 6번 — ps와 kill

ps (ps, "피에스")는 지금 일하고 있는 프로세스 목록을 한 번에 보여줘요. top과 비슷한데, top은 실시간이고 ps는 한 번 찍은 사진이에요. kill (kill, "킬")은 그 중 한 친구를 멈추게 해요.

> ▶ **같이 쳐보기**
>
> ```bash
> ps aux | head -20
> ```

aux는 옵션 묶음이에요. 모든 사용자(a)의 모든 프로세스(x)를 자세히(u). 결과가 너무 많으니까 `| head -20`으로 위 20줄만 봐요. 파이프와 head는 자주 쓰는 짝이에요.

kill은 보통 PID라는 번호로 멈춰요. `kill 12345`. 12345는 ps에서 봤던 그 친구의 PID예요. 멈추기는 위험해서 오늘은 시연 안 해요. 7교시 이후에 진짜 필요할 때 다시 봅니다.

## 9. 7번 — ifconfig와 ipconfig

이름이 비슷한데 다른 친구예요. macOS에는 둘 다 있어요. 리눅스에는 ifconfig만 있어요(요즘은 `ip` 명령으로 대체되는 추세).

> ▶ **같이 쳐보기**
>
> ```bash
> ipconfig getifaddr en0
> ifconfig en0 | head -10
> ```

`ipconfig getifaddr en0`는 우리의 와이파이 IP 주소 한 줄. `ifconfig en0`는 와이파이 카드의 자세한 정보 (MAC 주소, IP, 상태 등). en0가 비어 있으면 en1, en2를 시도해 보세요. 어떤 맥은 이더넷이 en0이고 와이파이가 en1이에요.

## 10. 8번 — ping과 curl

ping (ping, "핑")은 다른 컴퓨터한테 "있어?" 하고 물어봐요. curl (curl, "컬")은 다른 컴퓨터한테 진짜 데이터를 한 번 가져와 봐요.

> ▶ **같이 쳐보기**
>
> ```bash
> ping -c 3 google.com
> curl -s -o /dev/null -w "%{http_code}\n" https://google.com
> ```

ping의 -c 3은 세 번만 시도하고 멈추라는 뜻이에요. 안 그러면 무한히 가요. curl의 옵션은 좀 복잡한데, 짧게 말하면 "구글에 한 번 다녀와서 응답 코드만 알려줘"라는 뜻이에요. 200이 뜨면 성공. 두 해 후에 백엔드 만들 때 정말 자주 쓸 명령이에요.

## 11. 9번 — date와 uptime

date (date, "데이트")는 지금 시각, uptime (uptime, "업타임")은 우리 맥이 켜진 지 얼마나 됐는지.

> ▶ **같이 쳐보기**
>
> ```bash
> date
> uptime
> ```

date는 그냥 지금 시각. uptime은 "켜진 지 며칠, 평균 부하" 같은 정보. 평균 부하(load average) 숫자가 1.0 넘으면 시피유가 바쁜 상태예요. 6교시에 다시 봐요.

## 12. 10번 — man, --help, history

마지막은 명령어가 아니라 "모를 때 어디서 찾나"예요. 이게 진짜 중요해요.

> ▶ **같이 쳐보기**
>
> ```bash
> man ls
> ls --help 2>&1 | head -20
> history | tail -20
> ```

man (man, "맨")은 매뉴얼이에요. `man ls`라고 치면 ls의 사용법이 책처럼 떠요. q로 나와요. macOS의 일부 명령은 `--help`도 지원해요. 짧게 사용법만 보여줘요. 마지막 history는 우리가 최근에 친 명령 목록이에요. "어, 아까 그 명령 뭐였더라" 할 때 history로 찾아요.

## 👨‍💻 개발자 노트

> - 의도적으로 깊이를 얕게 유지. 각 명령의 본격 학습은 Ch 006(터미널 마스터)에서 진행. 본 카탈로그는 "두 해 동안 다시 만날 친구들의 얼굴 한 번 보기" 목적.
> - `top`/`ps`는 Ch 002 OS 챕터의 프로세스/스레드 본격 학습으로 직결. PID 개념을 여기서 한 번 노출하는 것이 OS 챕터 진입을 부드럽게 합니다.
> - `curl` 한 줄은 Ch 003 네트워크, Ch 030 HTTP, Ch 052 Flask API 디버깅, Ch 110 Lambda 호출 시험에서 거듭 회수됩니다.
> - `ifconfig`는 macOS·구리눅스 명령. 최신 리눅스는 `ip addr`. Ch 003에서 둘의 차이와 마이그레이션을 다룹니다. 본 챕터에서는 "macOS 기준" 단서만.
> - `man`을 마지막에 강조한 것은 의도 — "외우지 말고 찾는 사람이 되라"는 1교시 약속의 행동 지침입니다.
