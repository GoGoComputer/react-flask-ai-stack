## 2. 위험도 신호등 — 30 명령어를 세 색깔로

명령어를 세 색깔로 나누는 신호등을 자경단이 쓰고 있어요. Ch005 H4에서 git 명령어로 본 적 있으시죠. 셸 명령어도 같은 신호등을 적용해요.

**🟢 초록**은 read-only 명령어예요. 정보를 읽기만 하고 본인 노트북엔 아무 변화도 안 줘요. 사고 칠 확률 0%. ls, pwd, cat, ps, find 같은 게 초록이에요. 30개 중 17개가 초록이에요. 본인이 매일 만지는 명령의 절반 이상이 절대 안 망가져요.

**🟡 노랑**은 본인 노트북에만 변화를 주는 명령어예요. 파일 만들기, 옮기기, 복사하기. 본인이 실수해도 본인 노트북만 영향받고, 보통 복구 가능해요. mkdir, cp, mv, touch, sed 같은 게 노랑이에요. 8개 정도. 1초 안 호흡, 그냥 신중히 치세요.

**🔴 빨강**은 되돌리기 어려운 명령어예요. 한 번 치면 끝. rm, kill -9, dd, chmod -R 777 같은 다섯 개. 빨강 앞에서는 1초 호흡. 정말로 그 명령을 치고 싶은지 자기한테 물어봐요. 1초가 평생 후회를 막아줘요.

30개 중 빨강은 5개뿐이에요. 25개는 마음 편히. 다섯 개 앞에서만 정신 차려요. 그게 자경단의 안전 비결이에요. 외우려 마세요. 그냥 빨간 글자 보면 손가락이 멈추도록 훈련하세요. 6주면 자동으로 멈춰요.

## 4. 첫째 무리 — 파일 만지는 여덟 손가락

본인이 셸을 켜고 가장 먼저 치는 게 보통 이 여덟 개 중 하나예요. ls, cd, pwd, mkdir, cp, mv, rm, touch. 자경단 다섯 명이 매일 1,000번씩 치는 손가락이에요.

**ls**부터 가요. 디렉토리 안에 뭐가 있는지 보여 줘요. 옵션 6개를 외워 두면 본인 5년 인생이 가벼워요.

> ▶ **같이 쳐보기** — ls의 여섯 변주
>
> ```bash
> ls               # 단순 목록
> ls -l            # 상세 (권한·크기·시간)
> ls -la           # 숨은 파일 포함
> ls -lh           # 사람이 읽기 좋은 크기 (1.2K·5.3M)
> ls -lt           # 최근 수정 순
> ls -lS           # 크기 순
> ```

여섯 변주 중 본인이 매일 가장 자주 쓰는 건 `ls -la`. 그래서 자경단은 .zshrc에 `alias ll="ls -la"` 또는 `alias ll="eza -alh --git"`을 박아 둬요. 본인도 박으세요. 1초가 0.5초로 줄어요.

**cd**는 디렉토리 이동. `cd /tmp`로 절대 경로, `cd test`로 상대 경로, `cd ~`로 홈, `cd -`로 직전 디렉토리, `cd ..`로 한 단계 위. 다섯 가지가 본인 손가락에 박혀요.

**pwd**는 현재 위치 알려주는 명령. "Print Working Directory". 짧지만 매일 만나요. starship 프롬프트가 자동으로 보여 주지만 명시적으로 확인하고 싶을 때 한 줄.

**mkdir**은 디렉토리 만들기. `mkdir test` 한 줄. 깊은 트리를 한 번에 만들고 싶으면 `-p` 옵션. `mkdir -p a/b/c/d` 하면 네 단계가 한 번에. 자경단 미니가 자주 써요.

**cp**는 복사. `cp a.txt b.txt`로 파일 복사, `cp -r src/ dest/`로 디렉토리 통째 복사. `-r`이 recursive (재귀)의 약자예요.

**mv**는 이동 또는 이름 변경. 진짜로 이름 변경과 이동이 같은 명령이에요. `mv a.txt b.txt`는 이름 바꾸기, `mv a.txt /tmp/`는 옮기기. 같은 mv가 두 일 다 해요.

**rm**은 빨강이에요. 삭제. **1초 호흡**. `rm a.txt`는 안전, `rm -rf 폴더`는 위험. 특히 `rm -rf /` 절대 치지 마세요. 인터넷 농담이지만 진짜로 본인 노트북이 폭발해요. `-r` 옵션 앞에서는 항상 한 번 멈춰요.

자경단의 안전 alias 한 줄을 알려 드릴게요.

```bash
alias rm="rm -i"   # 매번 확인 묻기
```

이 한 줄을 .zshrc에 박으세요. rm 칠 때마다 "정말 지울까요? (y/n)" 물어봐요. 5초 귀찮지만 5년 안전.

**touch**는 빈 파일 만들기 또는 timestamp 갱신. `touch new.txt`는 빈 파일 생성, `touch existing.txt`는 시간 업데이트. 이 두 일을 같은 명령이 해요.

여덟 손가락. 본인이 매일 90% 시간을 이 여덟 개와 함께 보내세요. 외우려 마세요. 매일 쓰면 박혀요.

## 5. 둘째 무리 — 검색하는 다섯 손가락

파일 만지는 여덟 명 다음으로 자주 만나는 게 검색하는 다섯 명이에요. find, grep, rg, fd, which.

**find**는 파일 찾기 표준 도구예요. 50년 된 도구. 사용법은 한 가지 패턴.

```bash
find <어디서> <조건>
```

`find . -name "*.md"`. 현재 폴더에서 .md 파일 찾기. `find ~ -size +100M`. 홈에서 100MB 넘는 파일. `find /tmp -mtime -1`. /tmp에서 24시간 이내 수정된 파일. 옵션이 100개쯤 있는데 본인이 자주 쓰는 건 -name, -size, -mtime 셋이에요.

H1에서 본 그 한 줄 기억하시죠. `find ~ -type f -size +100M -exec ls -lh {} \; 2>/dev/null | sort -k5 -hr | head -5`. 이 한 줄 안에 find가 있어요. 100MB 넘는 파일 찾아서 ls로 정보 뽑아서 정렬해서 5개만. 본인이 이제 한 단어씩 읽을 수 있어요.

**grep**은 텍스트 검색이에요. 파일 안에서 특정 단어 찾기. `grep "TODO" *.md`는 .md 파일들에서 TODO라는 단어 찾기. 옵션 다섯 개.

```bash
grep -i "todo" *.md      # 대소문자 무시
grep -r "TODO" .         # 재귀 (하위 폴더까지)
grep -n "TODO" file.md   # 줄 번호 같이
grep -v "TODO" file.md   # TODO 없는 줄만
grep -c "TODO" *.md      # 매치 개수
```

자경단이 매일 쓰는 grep 패턴이에요. 다섯 개 외우시면 충분.

**rg** (ripgrep)은 grep의 진화 버전이에요. brew로 깔았죠. 100배 빨라요. 자경단 표준은 매일 grep 대신 rg를 쓰는 거예요. 사용법 거의 같아요.

```bash
rg "TODO"              # 현재 폴더 모든 파일에서 TODO
rg -i "todo"           # 대소문자 무시
rg --type md "TODO"    # .md 파일만
```

`grep -r .`이 5초 걸리는 폴더에서 `rg .`은 0.05초. 한 번 쓰시면 grep으로 못 돌아가세요.

**fd**는 find의 진화 버전. 직관적이고 빠름. 사용법 짧아요.

```bash
fd ".md$"              # .md로 끝나는 파일
fd -t d test           # test라는 디렉토리만
fd -t f -e py          # .py 확장자 파일만
```

find보다 사용법이 절반쯤 짧아요. 자경단 표준.

**which**는 명령어가 어디서 오는지 알려줘요. H2에서 봤죠. `which git`, `which ls`, `which python3`. type도 비슷한 일을 해요. type은 alias·function·builtin도 알려줘요.

다섯 손가락. 검색의 다섯 명. 본인이 매일 5번 이상 부르는 친구들이에요.

## 6. 셋째 무리 — 텍스트 다루는 아홉 손가락

본인이 셸의 진짜 강점을 만나는 무리예요. 텍스트 처리. cat, bat, head/tail, less, wc, sort, uniq, sed, awk, jq. 아홉 손가락. 셸이 100배 강력해지는 비밀이에요.

**cat**은 파일 내용 출력. `cat file.md` 한 줄로 끝. 짧은 파일은 cat으로, 긴 파일은 less로.

**bat**은 cat의 색깔 강화. brew로 깔았죠. `bat file.md` 하면 색깔과 줄번호와 함께 보여줘요. 자경단은 alias로 `cat`을 `bat`으로 매핑하기도 해요.

**head**는 처음 N줄, **tail**은 마지막 N줄. `head -10 file.md`는 처음 10줄, `tail -10 file.md`는 마지막 10줄. tail에 `-f` 옵션을 주면 실시간으로 추가되는 줄을 따라가요. `tail -f /var/log/app.log`. 미니가 매일 서버 로그 볼 때 쓰는 한 줄.

**less**는 페이지 단위로 보기. `less file.md`로 들어가면 화살표로 스크롤, `q`로 나가기, `/단어`로 검색, `n`으로 다음 매치. vim의 단순 버전이에요.

**wc**는 줄·단어·글자 수 세기. `wc -l file.md`는 줄 수, `wc -w`는 단어 수, `wc -c`는 글자 수. 본인이 책을 쓸 때 매일 만나는 친구. 저도 이 강의 대본 쓰면서 매번 `wc -m`으로 글자 수 확인해요.

**sort**는 정렬. `sort file.txt`는 알파벳 순, `sort -r`은 역순, `sort -n`은 숫자 정렬, `sort -k 2`는 2번째 칼럼 기준. 옵션 네 개가 본인의 정렬 일을 다 해줘요.

**uniq**는 중복 제거. 단, **정렬된 입력**이 필요해요. `sort file.txt | uniq`이 표준 패턴. uniq 혼자 쓰면 연속된 중복만 제거.

**sed**는 stream editor. 스트림 안에서 글자 치환. `sed 's/old/new/g' file.md`. file.md의 old를 new로 모든 곳 치환. 자경단이 매일 쓰는 한 줄. 파일 안 바꾸고 결과만 출력. 직접 바꾸려면 `-i` 옵션 (단, macOS는 `sed -i ''`로 빈 문자 필요).

**awk**는 컬럼 처리 도구예요. CSV나 공백 구분 데이터에서 특정 컬럼만 뽑기. `awk '{print $1}' file.csv`는 첫 컬럼만, `awk -F, '{print $2}' file.csv`는 콤마 구분의 둘째 컬럼만. 진짜 강력해요.

**jq**는 JSON 처리. brew로 깔았죠. `cat data.json | jq .name`으로 name 필드만 뽑기, `jq '.users[].email'`로 모든 user의 email. 자경단이 API 응답 디버깅할 때 매일 만나요.

아홉 손가락. 매일 만나는 건 cat, head, tail, grep(이건 검색 무리지만 텍스트와 자주 짝). 주간으로 만나는 건 sed, awk, jq, sort, uniq. 텍스트 처리가 셸의 진짜 힘이에요.

자경단 미니의 매일 한 줄을 보여드릴게요. 서버 로그에서 ERROR만 골라서 시간 분포 보기.

```bash
grep ERROR /var/log/app.log | awk '{print $1}' | sort | uniq -c
```

한 줄에 grep, awk, sort, uniq 네 명이 다 있어요. 셸의 무기 조합이에요. 이런 한 줄을 본인이 5년 후엔 즉흥으로 짤 수 있어요. 오늘은 그림만.

## 7. 넷째 무리 — 프로세스 보는 세 손가락

본인 노트북 안에서 도는 프로그램들을 보는 손가락이에요. ps, top, kill.

**ps**는 프로세스 목록. 단순히 `ps`만 치면 본인 셸이 띄운 프로세스만 보여요. 시스템 전체를 보려면 `ps aux`. a는 모든 사용자, u는 사용자 정보, x는 터미널 없는 것까지.

**top**은 실시간 모니터. 본인 노트북의 CPU·메모리 사용량을 1초마다 갱신해서 보여줘요. `q` 누르면 나가기. **htop**이라는 더 친절한 버전이 있어요. brew로 따로 깔 수 있어요.

**kill**은 빨강. 프로세스 종료. `kill 1234`로 PID 1234 프로세스에 종료 신호. `kill -15`는 부드럽게 (정상 종료 요청), `kill -9`는 강제 종료. **kill -9 앞에서 1초 호흡**. 데이터 손실 가능성 있어요.

PID는 ps로 찾아요. `ps aux | grep node`로 node 프로세스 찾고, 첫 줄의 두 번째 칼럼이 PID. 그걸 kill에 넘기면 끝.

자경단의 응급 한 줄. 포트 3000을 점령한 프로세스 죽이기.

```bash
lsof -i :3000 | tail -1 | awk '{print $2}' | xargs kill -9
```

lsof로 포트 3000 사용자 찾고, awk로 PID 뽑고, xargs로 kill에 넘기기. 한 줄 5초. 이 한 줄을 본인 dotfile에 `kill-port` function으로 박으면 평생 자산이에요.

## 8. 다섯째 무리 — 네트워크 두 손가락

**curl**은 HTTP 요청 도구. API 디버깅의 표준. 

```bash
curl https://api.github.com/users/anthropic
curl -X POST -d '{"name":"자경단"}' https://api.example.com
curl -H "Authorization: Bearer token" https://api.example.com
curl -O https://example.com/file.zip   # 다운로드
```

GET·POST·헤더·다운로드를 한 도구로. 자경단 까미가 매일 100번 치는 도구.

**ssh**는 원격 접속. `ssh user@server.com`으로 서버에 들어가요. 들어간 후엔 그 서버의 셸에서 명령을 칠 수 있어요. 본인 노트북의 명령어가 그대로 작동해요. 자경단 미니가 매일 8시간 ssh 안에서 일해요.

ssh의 짝꿍 **scp**는 파일 복사. `scp local.txt user@server:~/`로 보내기, `scp user@server:~/file.txt .`로 받기. 단, 자경단은 요즘 scp보다 rsync를 더 자주 써요. 더 빠르고 안전해요.

ssh 키 셋업은 한 번 해 두면 평생 매번 비밀번호 안 쳐도 돼요. `ssh-keygen`으로 키 만들고, `ssh-copy-id user@server`로 서버에 등록. 한 번이면 끝.

## 9. 여섯째 무리 — 아카이브와 조합 두 손가락

**tar**는 압축·해제. 자경단의 표준 한 줄.

```bash
tar -czf backup.tar.gz folder/    # 압축 (gzip)
tar -xzf backup.tar.gz            # 해제
tar -tzf backup.tar.gz            # 내용만 보기
```

c=create, x=extract, z=gzip, f=file. 네 글자 외워 두면 평생 가요.

**xargs**는 조합의 무기예요. stdin을 다른 명령의 인자로 변환.

```bash
ls *.tmp | xargs rm                    # tmp 파일 다 삭제
find . -name "*.log" | xargs rm        # 로그 파일 다 삭제
echo "1 2 3" | xargs -n 1 echo         # 한 줄에 한 개씩
```

xargs가 진짜 강력해요. pipe로 받은 결과를 다음 명령의 인자로 변신. 자경단이 자주 쓰는 한 줄이에요.

자, 30개 다 만났어요. 6 무리, 평균 5명씩. 표 한 장에 깨끗이 누워 있어요.

## 11. 자경단 한 줄 자동화 다섯 가지

마지막으로 자경단 다섯 명이 매일 쓰는 한 줄 자동화 다섯 개를 보여드릴게요. 30개 명령어가 한 줄에 어떻게 조합되는지 그림이에요.

**1. 본인 맥의 가장 큰 파일 5개 (H1 그 한 줄)**

```bash
find ~ -type f -size +100M -exec ls -lh {} \; 2>/dev/null | sort -k5 -hr | head -5
```

find + ls + sort + head. 30개 중 4개의 조합.

**2. 폴더 내 .md 파일 개수**

```bash
find . -name "*.md" | wc -l
```

find + wc. 가장 단순한 한 줄.

**3. 가장 자주 쓴 명령어 10개 (history 분석)**

```bash
history | awk '{print $2}' | sort | uniq -c | sort -rn | head -10
```

history + awk + sort + uniq + head. 5개 조합. 본인이 평소 어떤 명령을 자주 치는지 알 수 있어요.

**4. 100개 파일 한 번에 이름 바꾸기**

```bash
for f in *.txt; do mv "$f" "${f%.txt}.md"; done
```

H1에서 봤던 그 마법 한 줄. for 루프 + mv. 5초.

**5. JSON API 응답에서 특정 필드 추출**

```bash
curl -s https://api.github.com/users/anthropic | jq .public_repos
```

curl + jq. 자경단 까미가 매일 백 번 만나는 한 줄.

다섯 개. 각각 30개 중 2-5개 조합이에요. 본인이 30개를 손에 쥐면 이런 한 줄 100개가 가능해져요. H5에서 자경단 다섯 명이 30분 동안 짜는 자동화 라이브를 보여드려요.

## 12. 모던 대체 도구 다섯 가지

마지막으로 옛 도구의 새 버전 다섯 개를 다시 정리할게요. 자경단의 표준이에요.

| 옛 도구 | 새 도구 | 차이 |
|---------|---------|------|
| `grep` | `rg` (ripgrep) | 100배 빠름, .gitignore 자동 무시 |
| `find` | `fd` | 사용법 짧음, 자동 색깔 |
| `cat` | `bat` | 색깔, 줄번호, git 통합 |
| `ls` | `eza` | 색깔, 트리 모드, git 상태 |
| `man` | `tldr` | 5줄 예시 (1000줄 매뉴얼 대신) |

다섯 개를 .zshrc에 alias로 박아 두세요.

```bash
alias grep="rg"
alias find="fd"
alias cat="bat"
alias ls="eza"
alias man="tldr"
```

본인이 옛 명령을 쳐도 새 도구가 뜨게. 단, 옛 도구도 손가락에 박혀 있어야 해요. 다른 사람 컴퓨터에서 일할 때 새 도구가 안 깔려 있을 수 있거든요.

## 👨‍💻 개발자 노트 (참고 — 비개발자는 그냥 넘기셔도 됩니다)

> - GNU coreutils vs BSD utils: macOS는 BSD 기반이라 `ls`, `sed`, `find`가 GNU와 다름. `brew install coreutils gnu-sed findutils`로 GNU 버전 설치 가능. 자경단은 호환성 위해 BSD 기본 사용.
> - rm 안전성: 시스템 trash 사용은 `brew install trash` 또는 `rmtrash` cask. `rm`을 trash로 alias 권장.
> - find -exec vs xargs: `find ... -exec cmd {} \;`는 파일 하나마다 cmd 호출 (느림). `find ... -exec cmd {} +`는 한 번에 다 (빠름). xargs는 인자 길이 제한 (`getconf ARG_MAX`로 확인).
> - grep과 rg 차이: rg는 .gitignore와 .ignore 자동 적용, 멀티스레드, PCRE2 정규식. 단, BSD/macOS의 grep과 호환은 안 됨. 옛 스크립트는 grep 유지.
> - kill 신호 종류: SIGTERM(15)=정상 종료 요청, SIGKILL(9)=강제, SIGHUP(1)=재시작, SIGINT(2)=Ctrl+C, SIGCHLD=자식 종료 알림. `kill -l`로 모든 신호 목록.
> - sed -i 차이: GNU `sed -i 'expr' file`, BSD `sed -i '' 'expr' file`. 빈 문자 인자 필수. 호환 위해 `sed -i.bak` (백업 만들기) 패턴 권장.
> - jq 강력 필터: `jq '.users[] | select(.age > 30) | .name'`. SQL 비슷한 파이프 처리. JSON 다루는 모든 자경단의 도구.
> - tar 옵션 정리: c=create, x=extract, t=list, z=gzip, j=bzip2, J=xz, v=verbose, f=file. `tar czvf` (압축), `tar xzvf` (해제), `tar tzvf` (보기).
> - xargs vs find -exec: 인자 변환 vs 파일 변환. xargs는 stdin 전체 처리, -exec는 파일 단위. 성능 차이는 -exec ... +가 xargs와 거의 동일.
> - 다음 H5 키워드: 자경단 5명·30분 시뮬레이션·ERROR 진단·CSV·JSON·자동화 스크립트·통합.
