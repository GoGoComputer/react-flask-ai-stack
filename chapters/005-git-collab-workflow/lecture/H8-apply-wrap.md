## 3. 본인의 첫 PR — 1주차 이야기

본인이 회사에 입사한 첫 주 월요일. 9시.

```bash
# 9:00 - 회사 저장소 clone
git clone git@github.com:company/site.git
cd site

# 9:30 - 사수 형이 첫 task 줌
# "header에 자경단 로고 추가"

# 10:00 - 본인의 첫 branch
git switch -c feature/cat-vigilante-logo

# 11:00 - 코드 수정
vim src/components/Header.tsx

# 11:30 - 첫 commit
git add src/components/Header.tsx
git commit -m "feat(ui): cat vigilante logo"

# 12:00 - 점심 (한 시간)

# 13:00 - push
git push -u origin feature/cat-vigilante-logo

# 13:30 - 첫 PR
gh pr create --draft --title "feat: 자경단 로고" --body "..."

# 14:00 - 사수 형 리뷰
# "PR 본문에 스크린샷 추가해 줘"
# (배움 1)

# 14:30 - 스크린샷 + 다시
gh pr edit --body "스크린샷 ..."

# 15:00 - 본인이 ready 표시
gh pr ready

# 15:30 - 까미가 approve
# 노랭이가 nit 코멘트 ("로고 크기 32px이 좋겠어요")
# (배움 2)

# 16:00 - 수정 + push
git add -p
git commit -m "fix: logo size 32px"
git push

# 16:30 - 다시 approve

# 17:00 - 본인이 squash merge
gh pr merge --squash --delete-branch

# 17:30 - 본인의 첫 main 진입
# 슬랙에 자동 알림: "Bonin님의 첫 PR 머지 ✅"
```

본인의 첫 하루. 8시간에 PR 한 건. 다섯 번 배움.
