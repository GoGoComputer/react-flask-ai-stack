## 3. 옛날 이야기 — 제가 처음 정규식을 본 그 날

옛날 이야기. 제가 처음 정규식을 본 날. 12년 전.

회사에서 1만 개 로그 파일에서 ERROR 메시지의 시간만 뽑아 달라는 부탁. 저는 for 루프와 substring으로 끙끙댔어요. 100줄 코드. 1시간.

사수 형이 와서 한 줄 보여줬어요.

```python
re.findall(r'\d{2}:\d{2}:\d{2}.*ERROR.*', log_text)
```

한 줄. 5초. 같은 결과.

저는 그 한 줄에 충격받았어요. `\d{2}`가 두 자리 숫자, `:`가 콜론, `.*`가 아무 글자. 영어 같은 패턴. 그날 이후 저는 정규식에 빠졌어요. 매일 한 패턴씩 외웠어요. 1년 후엔 정규식 마법사. 본인도 8시간 후 똑같이 돼요.

## 6. 네 친구 — str·f-string·re·regex

**str**. 기본 문자열 + 30+ 메서드.

```python
"hello".upper()
"hi there".split()
" trim ".strip()
"abc".replace("a", "z")
```

**f-string**. 포매팅의 정점.

```python
f"{name}는 {age:.2f}살"
```

**re 모듈**. 정규식 표준 라이브러리.

```python
import re
re.match(r'\d+', text)
re.findall(r'\w+', text)
re.sub(r'old', 'new', text)
```

**regex** (서드파티). re의 진화.

```bash
pip install regex
```

```python
import regex
```

자경단 표준은 `re`. regex는 특수.

## 7. 0.001초의 여행 — re.search 5단계

`re.search(r'\d+', text)` 한 줄 실행.

**1. 패턴 컴파일**. 정규식을 NFA/DFA로.

**2. 텍스트 스캔**. 시작부터 매칭 시도.

**3. 매칭 발견**. 첫 매치 위치.

**4. Match 객체 생성**. .group(), .start() 등.

**5. 반환**.

5단계. 0.001초. 컴파일된 패턴은 캐싱.

## 10. 정규식 50년

1997년. PCRE (Perl Compatible Regular Expressions).

2000년. Python re 모듈 표준화.
